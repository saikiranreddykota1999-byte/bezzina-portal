## Before Coding

1. **Analyze** — Read the repository; map architecture, patterns, and affected files.
2. **Plan** — Break work into milestones; estimate effort; identify risks; list files to change; create a testing checklist.
3. **Explain** — Present the implementation plan before writing code.
4. **Implement** — Minimal, focused diffs; reuse existing abstractions.
5. **Test** — Unit, integration, security, accessibility, regression, edge cases, performance.
6. **Review** — Self-review against security, performance, and quality checklists below.
7. **Summarize** — Report what changed, why, risks, and follow-ups.

## Architecture Rules

- Prefer Server Components; use Client Components only when necessary.
- Prefer Server Actions for mutations.
- Keep components under 300 lines; functions under 50 lines.
- Never modify unrelated files.
- Never expose secrets.
- Never trust client input; always validate requests server-side.
